面向 AI Agent 的本地结构化命令行

AgentShell 的当前定位是 Codex/Claude Code/Cursor 等 coding agent 的本地插件与 CLI 辅助层： 把原本给人看的长日志、搜索输出和中间步骤，压缩成 agent 可直接消费的 JSON，减少 token、减少命令轮次，并在支持的失败测试场景里自动完成诊断、保守修复、验证和回滚指引。

PM 视角结论

短期价值已经明确

当前不是单纯“更好看的 CLI”，而是把 agent 使用终端的单位成本降下来。核心收益集中在 token 节省、命令轮次减少、失败测试修复链路标准化。

下一阶段要证明外部泛化

现在最需要的是把 adapter trial 跑到更多真实 host 和真实 repo 上，形成 Codex/Claude/Cursor 横向对比，而不是继续只堆功能。

速度收益要分层表达

AgentShell 自身命令很轻，但总耗时常被目标项目测试套件主导。所以对外应强调“减少 agent 轮次与输出成本”，速度指标作为同场景比较。

产品路线应继续插件优先

Codex 插件路径已经可验证、可打包、可自检。GitHub Release v0.25.3 已发布，远端下载的二进制、ZIP、校验文件和审计报告均已验证。外部用户证据是发布后的可选学习通道，不再阻塞 V1.0。MCP 暂不进入主线。

注意事项

Token 是基于 ceil(chars / 4) 的估算，不是模型 tokenizer 的精确值；duration 是本机命令耗时，不能直接承诺为跨机器 SLA。 Dashboard 的 351,219 tokens、12.371s 和 83% exact attribution 是本机快照；真实项目 smoke 主要证明低成本 pass-through，不等同于真实 bug repair 泛化已完全解决。当前最强 repair 证据仍来自 17 类 checked-in repair fixtures。
